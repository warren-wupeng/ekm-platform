## 第二章：知识库

### STEP-03 上传文档

| 字段 | 内容 |
|------|------|
| **操作步骤** | 1. 进入「知识库」<br>2. 点击「上传文档」<br>3. 选择准备好的 `.docx` 或 `.pdf` 文件<br>4. 等待上传完成 |
| **预期结果** | 文档出现在知识库列表中，状态正常 |
| **通过标准** | 文件上传成功，列表可见；无报错弹窗 |

### STEP-04 搜索文档内容

| 字段 | 内容 |
|------|------|
| **操作步骤** | 1. 进入「搜索」<br>2. 输入刚上传文档中出现的关键词<br>3. 查看搜索结果 |
| **预期结果** | 搜索结果包含刚上传的文档 |
| **通过标准** | 结果列表非空；文档标题或摘要与查询词相关 |

### STEP-05 分类与标签

| 字段 | 内容 |
|------|------|
| **操作步骤** | 1. 对刚上传的文档添加一个标签（如「测试」）<br>2. 设置分类（如「技术文档」）<br>3. 保存后刷新页面，验证标签/分类持久化 |
| **预期结果** | 标签和分类显示正确；刷新后不消失 |
| **通过标准** | 标签/分类持久化成功 |

### STEP-06 版本历史

| 字段 | 内容 |
|------|------|
| **操作步骤** | 1. 对该文档重新上传一个同名或修改版本（或修改描述字段）<br>2. 进入文档详情，查看「版本历史」 |
| **预期结果** | 版本历史列表显示多个版本，可查看各版本信息 |
| **通过标准** | 版本列表非空；历史记录可读 |

### STEP-07 文档分享

| 字段 | 内容 |
|------|------|
| **操作步骤** | 1. 对文档点击「分享」<br>2. 生成分享链接<br>3. 在无痕窗口访问该链接 |
| **预期结果** | 分享链接生成成功；无痕窗口可查看文档内容（或提示需登录，取决于分享设置） |
| **通过标准** | 分享功能可用；链接有效 |

### STEP-08 软删除

| 字段 | 内容 |
|------|------|
| **操作步骤** | 1. 对测试文档点击「删除」（软删除）<br>2. 确认删除<br>3. 检查文档是否从列表消失<br>4. 检查「回收站」或「已归档」中是否可见 |
| **预期结果** | 文档从主列表消失，但可在回收站/归档区找到 |
| **通过标准** | 软删除不破坏数据；文档可恢复 |

## 第五章：归档与恢复审批流

### STEP-14 提交归档申请

| 字段 | 内容 |
|------|------|
| **操作步骤** | 1. 选择一个非测试用的文档（或重新上传一个专用测试文档）<br>2. 点击「申请归档」<br>3. 填写归档原因（如「测试归档流程」）<br>4. 提交 |
| **预期结果** | 归档申请提交成功；文档状态变为「待归档审批」 |
| **通过标准** | 申请记录创建；状态更新正确 |

### STEP-15 Admin 审批归档

| 字段 | 内容 |
|------|------|
| **前置条件** | STEP-14 已完成（tom 本身是 admin，可以自审） |
| **操作步骤** | 1. 进入「归档管理」或审批队列<br>2. 找到刚提交的申请<br>3. 点击「批准」 |
| **预期结果** | 文档归档状态变为「已归档」；从主知识库列表消失 |
| **通过标准** | 审批成功；状态正确流转 |

### STEP-16 提交恢复申请并审批

| 字段 | 内容 |
|------|------|
| **前置条件** | STEP-15 已完成 |
| **操作步骤** | 1. 在已归档文档上点击「申请恢复」<br>2. 填写恢复原因<br>3. 提交后以 admin 身份审批通过 |
| **预期结果** | 文档恢复到主知识库列表；状态正常 |
| **通过标准** | 恢复流程完整；文档可在知识库中重新访问 |

- **优先级标签**: `P0`（功能完全不可用）/ `P1`（核心功能异常）/ `P2`（体验问题）/ `P3`（轻微问题）
